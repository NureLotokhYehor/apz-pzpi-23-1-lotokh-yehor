1 МЕТА РОБОТИ
Метою лабораторної роботи є розробка мобільного застосунку для платформи Android (на мові Kotlin), який є частиною системи «VetMonitor». Застосунок має забезпечувати взаємодію з серверною частиною через REST API для відображення стану пацієнтів та отримання push-сповіщень про критичні показники тварин.
2 ХІД РОБОТИ
У ході виконання роботи було реалізовано клієнтську частину системи VetMonitor для мобільних пристроїв. Застосунок інтегрується з раніше розробленим бекендом на Node.js та забезпечує мобільність ветеринарного персоналу.
2.1 Опис інженерних рішень
Для розробки було обрано архітектурний патерн MVVM (Model-View-ViewModel), що забезпечує розділення логіки обробки даних та інтерфейсу. Взаємодія з REST API реалізована за допомогою бібліотеки Retrofit 2, а отримання push-сповіщень інтегровано через Firebase Cloud Messaging.
2.2 Графічні матеріали (Діаграми)
2.2.1 UML діаграма прецедентів (Use Case Diagram) Діаграма описує взаємодію ветеринарного лікаря з мобільною платформою, включаючи авторизацію, перегляд списку пацієнтів та отримання тривожних сигналів.

import retrofit2.http.GET
import retrofit2.http.Path
import retrofit2.Response

interface VetApiService {
    // Отримання списку всіх пацієнтів клініки
    @GET("api/patients")
    suspend fun getPatients(): Response<List<Patient>>

    // Отримання поточних вітальних показників конкретної тварини
    @GET("api/vitals/{id}")

}

4 ВИСНОВКИ
У результаті виконання лабораторної роботи №2 було розроблено мобільний застосунок системи VetMonitor на базі ОС Android. Було спроєктовано архітектуру застосунку за стандартом MVVM та реалізовано механізм взаємодії з центральним сервером через RESTful API. Використання Kotlin та Retrofit дозволило створити стабільне рішення для оперативного моніторингу стану тварин. Створені UML-діаграми підтверджують відповідність застосунку функціональним вимогам, описаним у Vision & Scope.
5 КОНТРОЛЬНІ ЗАПИТАННЯ
1.	Яка перевага використання Kotlin для Android у порівнянні з Java? Kotlin забезпечує Null-safety, має більш лаконічний синтаксис та повну підтримку корутин для асинхронних запитів до мережі.
2.	Навіщо у звіті потрібна Component Diagram? Вона дозволяє візуалізувати фізичну організацію та залежності між програмними модулями системи, що важливо для архітектурного аналізу.
3.	Як реалізовано зв'язок мобільного застосунку з IoT-датчиками? Прямий зв'язок відсутній; застосунок отримує дані через проміжний сервер (Backend), який агрегує інформацію від сенсорів.
