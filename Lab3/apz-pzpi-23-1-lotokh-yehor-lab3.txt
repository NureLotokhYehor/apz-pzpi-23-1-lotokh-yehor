1 МЕТА РОБОТИ
Метою лабораторної роботи є розробка фронтенд-частини системи VetMonitor, що забезпечує роботу користувачів у веб-режимі. Завдання включає створення окремих інтерфейсів для ветеринарів та адміністраторів, реалізацію механізмів інтернаціоналізації (локалізації) та функцій адміністрування системи, таких як керування даними та резервне копіювання.
2 ХІД РОБОТИ
У межах роботи було реалізовано веб-застосунок, який є центральним вузлом для моніторингу стану пацієнтів та керування інфраструктурою клініки.
2.1 Опис прийнятих інженерних рішень
1) Технологічний стек: Для розробки використано бібліотеку React.js та фреймворк Tailwind CSS для створення адаптивного інтерфейсу. 
2) Архітектурний підхід: Застосунок побудовано за компонентно-орієнтованою архітектурою, де кожен елемент інтерфейсу (графік, таблиця, форма) є незалежним модулем. 
3) Локалізація: Використано бібліотеку i18next для підтримки української та англійської мов. Система автоматично адаптує формат дати (ДД.ММ.РРРР для України та ММ/ДД/РРРР для США/Великої Британії) та порядок сортування текстових значень. 
4) Рольова модель: Реалізовано два типи доступу. 
•	 Перший користувач (Ветеринар): Перегляд вітальних даних у реальному часі та керування профілями тварин.
•	 Адміністратор: Повний доступ до керування обліковими записами, експорту/імпорту даних у форматі JSON/CSV та створення резервних копій бази даних.
2.2 Опис взаємодії компонентів
Веб-застосунок взаємодіє з бекендом за допомогою REST API (бібліотека Axios). Компоненти дашборду отримують оновлені дані від сервера, які бекенд своєю чергою зчитує з IoT-сенсорів через протокол MQTT або HTTP.

2.5 Опис інтерфейсу веб-застосунку
Нижче наведено графічне представлення реалізованих модулів системи VetMonitor, що демонструють роботу ветеринарного та адміністративного персоналу.
2.5.1 Панель загального моніторингу пацієнтів На Рисунку 3 представлено основний робочий стіл ветеринара. Реалізовано систему карток пацієнтів із кольоровою індикацією стану (Stable/Critical). Система автоматично оновлює вітальні показники (пульс, температура) у реальному часі для всіх активних пацієнтів стаціонару.

 
Рисунок 5 - Інтерфейс моніторингу стану тварин у реальному часі

2.5.2 Детальна аналітика стану пацієнта 
При переході до профілю конкретної тварини (Рисунок 4) відображається розширена аналітика. Реалізовано візуалізацію історії серцевого ритму та стабільності температури за допомогою інтерактивних графіків. У верхній панелі продемонстровано роботу модуля локалізації (перемикач UA/EN), що адаптує всі текстові значення та формати вимірювання під обраний регіон.
 
Рисунок 6 - Картка пацієнта з історичними даними та модулем локалізації

2.5.3 Модуль управління записами (Appointments)
 Для координації роботи клініки реалізовано календарне планування (Рисунок 7). Ветеринари можуть переглядати графік на день, тиждень або місяць, а також фільтрувати візити за типами (вакцинація, екстрена допомога, грумінг). Кожен запис містить детальну інформацію про власника та причину звернення.

 
Рисунок 7 - Інтерфейс менеджменту клінічних візитів

2.5.4 Адміністративна панель та аналітика На Рисунку 6 представлено інтерфейс для адміністратора системи (Admin Portal). Модуль дозволяє відстежувати загальну продуктивність клініки: кількість пацієнтів, фінансові показники та динаміку реєстрацій. Також реалізовано функцію експорту фінансових звітів та даних пацієнтів у формат CSV для подальшого аналізу або створення резервних копій.

3 ВИСНОВКИ
У результаті виконання лабораторної роботи №3 було розроблено веб-застосунок для системи VetMonitor. Реалізовано дворівневу систему доступу (Користувач/Адміністратор) та механізми локалізації інтерфейсу для підтримки української та англійської мов. Застосунок забезпечує повний цикл адміністрування системи, включаючи імпорт/експорт даних та резервне копіювання, що відповідає вимогам до архітектури сучасних корпоративних рішень.
4 КОНТРОЛЬНІ ЗАПИТАННЯ
1) Чим відрізняється інтернаціоналізація від локалізації? Інтернаціоналізація - це технічна підготовка коду до підтримки багатьох мов, а локалізація - це безпосередній переклад та адаптація контенту під конкретний регіон (дати, валюти).
 2) Які функції повинен виконувати адміністратор у веб-застосунку? Керування обліковими записами, управління системними даними, створення бекапів та експорт/імпорт налаштувань.
 3) Навіщо використовувати компонентний підхід у React? Це дозволяє перевикористовувати код, спрощує тестування та полегшує підтримку великих архітектурних систем, таких як VetMonitor.
