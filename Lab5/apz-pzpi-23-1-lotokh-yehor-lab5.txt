Харків 2026
 
1 МЕТА РОБОТИ
Метою фінальної роботи є систематизація результатів розробки системи VetMonitor за два семестри та проведення підсумкової презентації готового продукту. Робота включає аналіз відповідності реалізованих функцій початковим вимогам, підготовку демонстраційних матеріалів та формування відео-демонстрації цілісної екосистеми моніторингу здоров’я тварин.
2 ХІД РОБОТИ
2.1 Підсумковий опис проекту VetMonitor
Система VetMonitor є комплексним архітектурним рішенням для ветеринарних закладів, що об'єднує IoT-технології, хмарні обчислення та багаторівневий інтерфейс користувача. Проєкт успішно вирішує проблему безперервного контролю вітальних показників пацієнтів у стаціонарних відділеннях, дозволяючи мінімізувати ручну працю персоналу та підвищити точність медичних записів. Протягом розробки було реалізовано бекенд на Node.js, веб-дашборд на React та Android-застосунок на Kotlin, що взаємодіють через єдине API середовище.
2.2 Статус реалізації функцій системи
У ході розробки було повністю виконано наступні ключові функції, передбачені документом Vision & Scope:
2.2.1	Реалізовано модуль автентифікації користувачів з підтримкою Google API Auth та рольовою моделлю доступу.
2.2.2	Розроблено веб-панель для ветеринарів, що дозволяє виконувати всі CRUD-операції над профілями пацієнтів та переглядати історію показників у реальному часі.
2.2.3	Створено Android-застосунок, який забезпечує отримання миттєвих push-сповіщень про критичні зміни стану тварин, що знаходяться під наглядом.
2.2.4	Впроваджено підтримку локалізації інтерфейсу (UA/EN) та формати дати відповідно до регіону користувача.
2.2.5	Реалізовано механізми адміністрування системи, включаючи експорт даних у CSV та створення резервних копій бази даних.
2.2.6	Додано архітектурну можливість горизонтального масштабування бекенда для роботи під високим навантаженням.
2.3 Демонстраційні матеріали та відео-демо
Для фінального звіту підготовлено презентаційні слайди, що описують бізнес-цінність продукту, його технологічний стек та архітектурні переваги. Також підготовлено відеозапис тривалістю 5 хвилин, де продемонстровано реальний сценарій використання: від підключення віртуального датчика до тварини до отримання тривожного сигналу на смартфоні ветеринара при виявленні аномалії. 3 ВИСНОВКИ
За результатами двох семестрів роботи було створено повноцінну програмну систему, що відповідає сучасним стандартам розробки ПЗ. Проект VetMonitor довів свою життєздатність як інструмент для автоматизації процесів у ветеринарії. Було закріплено навички роботи з повним циклом розробки: від формування бізнес-вимог та Vision & Scope до налаштування масштабованих хмарних інфраструктур
